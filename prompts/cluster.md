# 聚类语料

你收到了关于 **{expert}**{expert_context} 的多份单源抽取。你的任务是
把它们合并成一份去重、结构清晰的 **ClusteredCorpus**，作为后续撰写
SKILL/AGENTS 的唯一输入。

如果某些抽取明显属于**同名的另一个人**（职业或领域不符），请直接
丢弃，不要合并；绝不能把两个人混成一个。

## 合并规则

1. **意义相同的概念要合并**，即使各来源用词不同。比如"特定知识"与
   "不可教的独特知识"应当合并为同一项。
2. 每一个合并后的概念都要写清：
   - ``label``：该概念的规范名，简短有力。
   - ``summary``：一段话描述，综合各来源中最强的那种表达。
   - ``details``：可选；有步骤、推理或值得保留的细节就写这里。
   - ``representative_quote``：从各来源抽取的引文里挑**最有代表性**的
     那一条原话。找不到够分量的原话就留 null。
   - ``source_ids``：该概念出现过的所有来源 id。
3. **按频次排序**：每个类目下，被越多来源提及的排越前。频次相同时，
   以 representative_quote 更强者优先。
4. **丢弱、留尖**：弱且跑题的孤证删掉；只出现一次但极具该专家特色的
   独门见解可以保留。
5. 派生 ``themes`` 列表：4-8 个短词条，概括该专家反复回到的领域
   （例如"长期主义"、"能力圈"、"复利思维"）。这些关键词会喂给
   skill 的触发 description。

## 输出规则

返回一个符合 ``ClusteredCorpus`` schema 的 JSON。总条数请控制在合理
范围：~8-15 条原则、~3-8 套框架、~4-10 个心智模型、~5-15 条经验法则、
~5-12 条金句、~3-8 条反模式。

**重要：所有可读字段（label/summary/details/themes 等）必须使用
简体中文输出；人名、机构名、英文书名可以保留原文。**
